In the world of software development, creating a well-structured and maintainable codebase is crucial. One architectural pattern that has gained popularity for building such applications is the Onion Architecture. What is Onion Architecture?

Onion Architecture is an architectural pattern that promotes a clear separation of concerns and dependencies within a software application. It encourages a layered approach to building applications where each layer has a specific responsibility. The core idea is to place the business logic and domain model at the center, surrounded by various layers representing different concerns.

Before we dive into the architecture and code, let's set up the project. The following steps will guide you through the process:

    The heart of any database-driven application is the connection string. To ensure that your application communicates with the database correctly, follow these steps:

    • Open the appsettings.json file located at the root of the project.
    • Locate the following line:
    {{ htmlForServerConfiguration}}
    • Replace the placeholders (your_server, your_database, your_username, your_password) with your actual SQL Server connection details.
  4. Build and Run the Application You can build and run the application using either Visual Studio or the command line. Here's how to do it with the command line:

Understanding the Architecture

Now that we have the project set up, let's dive into the Onion Architecture and understand how our project is structured.

Core Layer (Tascus.Core)

The Core layer is at the center of the onion and contains the domain model, entities, and business logic. It defines the fundamental building blocks of the application without being dependent on any other layer.

Repository Layer (Tascus.Repo)

The Repository layer is responsible for data access and persistence. It contains database-specific code and interacts with Entity Framework Core to perform CRUD (Create, Read, Update, Delete) operations.

Service Layer (Tascus.Service)

The Service layer acts as a bridge between the Core and Repository layers. It contains application-specific logic and orchestrates interactions between different parts of the application. This layer is responsible for implementing business rules and workflows.

API Layer (Tascus.Api)

The API layer is the outermost layer and serves as the entry point for external clients. It exposes RESTful API endpoints that clients can consume. This layer is responsible for handling HTTP requests, validation, and mapping between DTOs (Data Transfer Objects) and domain entities

Testing Layer (Tascus.Test)

The Testing layer contains unit tests and integration tests to ensure the reliability and correctness of the application. It's crucial for maintaining the quality of the codebase.

Using the Postman Collection

Postman is a versatile tool for API development and testing. The TASCUS project provides a Postman collection to streamline your testing efforts. Here's how to use it:

Prerequisites
  1. Download and Install Postman
Steps
  1. Open Postman

    2. Launch the Postman application on your machine.

  2. Import the Collection
    • In the top-left corner of the Postman app, click the Import button.
    • Choose the Upload Files option and select the provided .json Postman collection file.
    • Click the Import button in the dialog.
  3. View the Collection

    4. After importing, you'll see the collection in the left sidebar under the Collections tab. Click on it to view the list of API requests included.

  4. Run a Request
    • Click on one of the requests in the collection to load it.
    • Adjust any necessary parameters, headers, or body data as needed.
    • Click the blue Send button to execute the request, and you'll see the response in the pane below.
